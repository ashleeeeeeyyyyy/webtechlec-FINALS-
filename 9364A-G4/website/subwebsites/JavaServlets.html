Java Servlets

  • Java objects intended to play the role of server components in client-server communications.
  • Handles client requests by generating responses to such requests.
  • Part of the Java EE specification

Servlet Lifecycle

  • INSTANTIATION -> constructor call
    • Creates an instance of the servlet
    • Invoked implicitly by servlet container
  • INITIALIZATION -> init() method
    • Invoked only once, intended for startup initialization code
    • (requires additional notes) 
      • public void init(ServletConfig conf) throws ServletException
  • REQUEST HANDLING -> service() method
    • Multiple client requests handled by a single multithreaded servlet instance
    • This processes the client's requests.
    • (requires additional notes) 
      • public void service(ServletRequest req, ServletResponse res)   
        throws ServletException, IOException
  • DESTRUCTION -> destroy() method
      (requires additional notes) 
      public void destroy()

HTTP Servlet

  • Java Servlet used to handle HTTP requests
  • Hosted in a web container
  • service() method call is routed to a doXXX() call, depending on the HTTP request method
      • doXXX()methods has 2 arguments:
    • HTTPServletRequest Used to access information from the request message
    • HTTPServletResponse Used to set information in the response message
